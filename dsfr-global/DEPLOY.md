# Deploy — DSFR Global

Stack: **GitHub** (source) → **Render** (API + web + Redis) → **Neon** (Postgres).

## 0. Push to GitHub

From the project root (`dsfr-global/`):

```bash
git init
git add .
git commit -m "DSFR Global: initial platform scaffold + deploy config"
git branch -M main
git remote add origin https://github.com/devdsfr/dsfr-global.git
git push -u origin main
```

> If the repo already has commits, use `git pull --rebase origin main` before pushing,
> or force with care: `git push -u origin main --force` (only if the repo is empty).

## 1. Neon — Postgres

1. Open https://console.neon.tech and create a project (e.g. `dsfr-global`).
2. In **Dashboard → Connection string**, copy the **pooled** connection string. It looks like:
   `postgresql://user:pass@ep-xxx-pooler.region.aws.neon.tech/dbname?sslmode=require`
3. Keep it — you'll paste it into Render as `DATABASE_URL`.

No manual SQL needed: the API runs an idempotent migration on startup and creates the `users` table automatically.

## 2. Render — Blueprint (one click for everything)

1. Open https://dashboard.render.com → **New → Blueprint**.
2. Connect the GitHub repo `devdsfr/dsfr-global`. Render reads `render.yaml` and proposes:
   - `dsfr-global-api` (Go web service, Docker)
   - `dsfr-global-web` (Angular static site)
   - `dsfr-global-redis` (Key Value / Redis)
3. Click **Apply**. `JWT_SECRET` is auto-generated and `REDIS_URL` is auto-wired.
4. One value is left blank on purpose — set it now:
   - On **dsfr-global-api → Environment**, set `DATABASE_URL` to your Neon pooled string (step 1).
5. Render builds and deploys. The API health check is `/healthz`.

## 3. Cross-wire the URLs

Render service URLs follow `https://<name>.onrender.com`. The blueprint assumes:
- API: `https://dsfr-global-api.onrender.com`
- Web: `https://dsfr-global-web.onrender.com`

If Render appended a suffix (name collisions add random chars), update **both**:
- **Frontend** → `frontend/src/environments/environment.prod.ts` → `apiUrl` → your real API URL, then commit/push (triggers rebuild).
- **API** → env var `ALLOWED_ORIGINS` → your real web URL (CORS).

## 4. Verify

```bash
curl https://dsfr-global-api.onrender.com/healthz
# {"service":"dsfr-global-api","status":"ok"}
```

Open the web URL, register an account, and log in.

## Notes / gotchas

- **Free plan cold starts**: the free API sleeps after inactivity; the first request after idle takes ~30–60s. Upgrade the API service to a paid instance to keep it warm.
- **Neon SSL**: keep `?sslmode=require` in the connection string. Use the **pooled** endpoint for serverless-style connections.
- **Rotating secrets**: `JWT_SECRET` is generated by Render; rotating it invalidates existing access tokens (users just log in again).
- **Rate limiter** is in-memory (per instance). Fine for a single free instance; move to a Redis-based limiter before scaling to multiple API instances.
- **Password reset** currently logs the token (LogMailer). Wire an SMTP/SES mailer before going live with real users.
